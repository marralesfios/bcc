Building from source

BCC depends on many in-house dependencies, variously being header-only and requiring GNU Make to build. But worry not! We have made a simple "package manager" in Python that will automatically download and build all of them for you. (And it's fully self-contained — it doesn't mess with your ~ or environment variables in any way.)

Note that we only support Linux builds for the time being. If you are on Windows, you may try building in WSL, but we can't guarantee it'll work.

Prerequisites

  • Python 3
  • GCC 16[1]
  • git
  • C/C++ libraries: stb, SDL3, harfbuzz, fontconfig, tbb. [2]

Building

  1. Download mpm.py. You don't have to put it into an empty folder, since it creates a `mpm` subdirectory to put all downloaded files in.
  2. Download and build the library components (click the command to copy): ./mpm.py cppp assembly glad sgl efficiency-bubble:bincode-backend efficiency-bubble:bincode-frontend
  3. Build the editor executable: . mpm/activate.sh # set environment variables for include and static library directories cd mpm/pkgs/efficiency-bubble_bincode-frontend make bin/test

That's it! Type bin/test into the terminal to launch the editor. (The first time start-up is a bit slower, since it needs to invoke GCC to compile a helper program. Don't worry, this doesn't need GCC 16, so the system GCC will do just fine.)

Note: this is very much still a test program; the editor executable must be run with a CWD of (directory of mpm.py)/mpm/pkgs/efficiency-bubble_bincode-frontend, or else it will fail! We will eventually address this in the real release.

1: Yes, you need GCC 16. Earlier versions WILL NOT work. Note that it doesn't have to be in your PATH as gcc/g++; you can build/download GCC 16 into another directory, and select it while running the package manager.

2:Most of them should be available in your distro's package manager; otherwise, you can probably find installation instructions from Google. As a last resort, get these libraries from source from the linked official repos and attempt to install them yourself — but I probably don't need to repeat why installing things into your system include and library directories without using your package manager is a bad idea.
